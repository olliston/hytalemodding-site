### PondFiller

Fills depressions in the terrain with the provided MaterialProvider.

The screenshots below show a few examples of ponds generated with this Prop.

![Example of PondFiller Prop1](/assets/official-documentation/worldgen/technical-hytale-generator/props/pondfiller-example-1.png)


![Example of PondFiller Prop2](/assets/official-documentation/worldgen/technical-hytale-generator/props/pondfiller-example-2.png)


You can configure the bounding box inside which it will identify and fill existing terrain depressions. The bounding box’s size and shape determine how large the resulting ponds can be.

The diagram below shows how the bounding box defines the outcome.

![Example of PondFiller bounding box](/assets/official-documentation/worldgen/technical-hytale-generator/props/pondfiller-bounding-box.png)


The ponds that fit inside that bounding box are filled with the MaterialProvider.

The BarrierBlockSet defines what the walls of the depressions are made of.

Important: The performance impact of this Prop can be relatively high depending on the size of its bounding box. Because of that, I recommend optimizing the size of its bounding box for each use case.

##### Parameters

<table><thead><tr><th width="151.09912109375">Name</th><th width="202.95050048828125">Type</th><th>Description</th></tr></thead><tbody><tr><td>Bounds</td><td>Bounds</td><td>The Prop's reach.</td></tr><tr><td>BarrierBlockSet</td><td>BlockSet asset slot</td><td>Defines the types of blocks that can make up the solid terrain, through which the pond material cannot seep.</td></tr><tr><td>FillMaterial</td><td>MaterialProvider asset slot</td><td>Determines the types of blocks used when filling up the ponds (for example, water).</td></tr></tbody></table>
